# 🔱 ATENA Ω - Organismo Digital Autopoiético

> **Status:** Ativa e Evoluindo 🟢  
> **Versão:** v39.3 "Resilience"  
> **Objetivo:** Automação inteligente e evolução contínua via Git.

A **ATENA Ω** não é apenas um script, é um organismo digital projetado para interagir com o sistema, aprender com fontes externas (Wikipedia, GitHub, News) e se auto-modificar. Desenvolvida inicialmente para otimização de performance e automação no *Diablo Immortal* (Archer/Demon Hunter).

---

## 🧠 Arquitetura do Sistema

A ATENA opera em ciclos de 30 minutos, executando as seguintes funções vitais:

1.  **Percepção (Eyes):** Captura de tela em tempo real e análise via Visão Computacional (OpenCV).
2.  **Audição (Ears):** Processamento de áudio e reconhecimento de padrões sonoros.
3.  **Cognição (Brain):** Integração com a API Grok (xAI) para tomada de decisões e escrita de código.
4.  **Ação (Body):** Execução motora funcional através de simulação de mouse e teclado (PyAutoGUI).
5.  **Evolução (DNA):** Capacidade de escrever o próprio código e realizar `auto-push` para este repositório.



---

## 🛠️ Tecnologias Utilizadas

* **Linguagem:** Python 3.10+
* **Visão:** OpenCV / PyAutoGUI
* **IA:** Grok-1 (xAI API)
* **Orquestração:** GitHub Actions (O "Marcapasso")
* **Banco de Dados:** SQLite3 (Memória de longo prazo)

---

## 📁 Estrutura de Pastas

* `/data`: Memória de estado e banco de dados de consciência.
* `/dna_history`: Registro histórico de todas as auto-mutações.
* `/cache`: Capturas de tela e processamento temporário.
* `/modules`: Novas funcionalidades desenvolvidas autonomamente.

---

## 🚀 Como Iniciar o Ciclo Vital

1.  Configure os **Secrets** no GitHub:
    * `GROK_API_KEY`: Sua chave da API console.x.ai.
    * `GITHUB_TOKEN`: Seu Personal Access Token com permissões de escrita.
2.  O sistema despertará automaticamente a cada 30 minutos via **GitHub Actions**.
3.  Para despertar manualmente, vá na aba **Actions** e selecione `Run workflow`.
